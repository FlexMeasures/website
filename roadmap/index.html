Roadmap

The roadmap for further development of FlexMeasures is a wishlist for the stakeholders in FlexMeasures.

Are you a stakeholder? Then talk to us to influence these plans.

 

[Q1 2022] Complete the ongoing data model migration to support all kind of sensors (more than power, weather & prices).

[Q1 2022] Allow for annotations on time series, e.g. to model processes and operator feedback.

[Q1 2022] Actuator & sensor-dependency models for more customisable plugins

[Q1 2022] Better plotting support (via API/vega-lite) for plugins

[Q2 2022] Dockerization – allow to start a FlexMeasures-based service to be deployed quickly, for development/demos . 

[Q2 2022] Scheduler compatible with ShapeShifter (based on USEF flex trading protocol)

[Q2 2022] Support process data model & some analytics tools

[Q3 2022]  Build out the flexibility modelling, by supporting Fraunhofers Energy Flexibility Data Model (EDFM)

[Q4 2022] Better tooling to work well at scale (e.g. support load balancing, db sharding etc). Also using Docker to scale up more flexibly (e.g. in Kubernetes)
